Grammar Feeder And Pool microservice

There will be two queues : 1. grammer-pool-queue 2. grammer-feeder-queue

grammer-pool-queue : This queue will be responsible for handling requests for grammar pool. The request and response dto will be more like key and value pair.

Currently both of these key value pairs are of string type. But value can be of custom type. like key can be birthday and its value can be in pool as xxx date

2. grammar-feeder-queue : This queue will be responsible for feeding grammar inside the pool. The dto will be having key, value and customDTO. The customDTO will be having name and date.

The Pool will be divided into two main parts:

1. common pool 2. private pool

the objetcive is to first feed grammar into the common pool, but some of the words will remain into the private pool, like keyword birthday will remain in common pool but its value will be '/' i.e an indication that its value will be in the private pool.

Strategy to find a word:

always try to first find the word inside the common pool, if not found then try to find it out in the private pool.

below are the cases which will come into picture:

1. keyword found in common pool return value. 2. keyword found in the common pool but the value is the pointer to the private pool.3. keyword not found in the common pool, try to find out in the private pool, if found then return. if not then some message.

keyword which will remian in common pool : before, after, hour, min, day, days, months, month, year, years
